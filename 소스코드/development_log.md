# 전염병 확산 시뮬레이터 개발 로그

## 버전 1.0: SIR 미분 방정식 모델 구현
- **요청사항:** 미분 방정식 기반의 전문적인 모델로 변경.
- **작업 내용:** `scipy`를 사용하여 SIR 모델의 미분 방정식을 계산하는 `sir_ode_model.py` 작성.

## 버전 2.0: GUI 구현 및 기능 확장 (SIRVD)
- **요청사항:** GUI 구현, 백신(V) 및 사망자(D) 상태 추가.
- **작업 내용:** `tkinter`를 사용하여 GUI 프로그램 `sir_ode_gui.py` 작성. 모델을 SIRVD로 확장.

## 버전 3.0: SEIRVD 모델 및 편의 기능 추가
- **요청사항:** 잠복기(E) 상태 반영, 바이러스 프리셋 기능 추가.
- **작업 내용:** 모델을 SEIRVD로 확장하고, `virus_presets.json`을 이용한 프리셋 기능 구현.

## 버전 4.0: UI 한글화 시도 및 실패
- **요청사항:** 프로그램 UI를 한글로 변경.
- **작업 내용:** `ui_labels_ko.json`을 이용한 국제화(i18n) 방식 적용.
- **발생 오류:** `tkinter`의 시스템 폰트 인식 문제로 글자 깨짐 현상 발생.

## 버전 5.0: 한글화 재시도 및 폰트 문제 해결
- **요청사항:** 폰트 깨짐 문제를 해결하고 한글 UI를 완성.
- **작업 내용:** `tkinter`와 `matplotlib`의 기본 폰트를 'Malgun Gothic'으로 명시적으로 지정하여 문제 해결.

## 버전 6.0: SEIRS 모델 업그레이드 및 UI/UX 개선
- **요청사항:**
    - 시간이 지나면 면역이 사라지는 현상(Waning Immunity)과 백신 돌파 감염(Breakthrough Infection)을 반영하여 모델의 현실성 증대.
    - '회복자'가 더 높은 비율로 백신을 맞는 행동 모델링.
    - 복잡해진 UI를 '기본'과 '상세' 설정으로 분리하여 사용성 개선.
    - 의미있는 시뮬레이션을 위한 '추천 초기 감염자 수' 가이드라인 제공.
- **작업 내용:**
    - **SEIRS 모델로 전환:** 회복자(R)가 다시 감염 가능(S) 상태로 돌아가는 로직 추가.
    - **돌파 감염 구현:** 백신 접종자(V)가 일정 확률로 다시 감염되도록 수정.
    - **차등 접종률 구현:** '회복자 백신 접종 보너스' 파라미터를 추가하여, 회복자(R) 그룹이 다른 그룹보다 높은 비율로 백신을 맞도록 미분 방정식 수정.
    - **초기 조건 개선:** 비현실적인 `E0 = I0 * 5` 로직을 폐기하고, 바이러스 특성에 기반하여 `E0`를 동적으로 계산하도록 변경.
    - **UI 리팩토링:** 상세 설정을 별도 창으로 분리.
    - **가이드라인 추가:** 전체 인구(N)에 비례하여 '추천 초기 감염자 수'를 계산하고 UI에 표시.

## 버전 7.0: 고급 면역 모델링 및 로직 개선
- **요청사항:** 자연 면역과 백신 면역의 특성을 독립적으로 설정하고, 잠복자도 백신을 맞도록 수정.
- **작업 내용:**
    - **독립적 면역 시스템:** 자연 면역과 백신 면역의 효과/지속기간을 독립적으로 설정하도록 파라미터 분리.
    - **잠복자 백신 접종:** 잠복 감염자(E)도 백신을 맞고 감염을 회피하는 로직 추가.

## 버전 8.0: 시나리오 기반 시뮬레이션 도입
- **요청사항:** 시간에 따라 바이러스 특성 및 외부 개입이 변하는 동적인 시뮬레이션 기능 추가.
- **작업 내용:**
    - **변이 바이러스 시나리오:** 시간에 따라 다른 특성을 가진 변이가 출현하는 `variants.json` 기능 구현.
    - **개입 시나리오:** 사회적 거리두기 등 시간에 따른 감염률 변화 `interventions.json` 기능 구현.
    - **UI 통합:** 기존의 상세 설정을 모두 시나리오 기반으로 통합.

## 버전 9.0: 사용성 개선
- **요청사항:** 복잡한 시나리오 기능들을 간단히 켜고 끌 수 있는 On/Off 기능 추가.
- **작업 내용:**
    - **시나리오 On/Off 기능:** 메인 UI에 '변이 시나리오 사용', '개입 시나리오 사용' 체크박스 추가.
    - **'상세 설정' 창 복구:** '변이 시나리오'를 끄면, 개별 파라미터를 직접 수정할 수 있도록 '상세 설정' 창과 관련 기능 복구.
    - **로직 분기:** On/Off 상태에 따라 `variants.json`을 사용하거나, '상세 설정'의 값을 사용하도록 `run_simulation` 로직 수정.
    - **코드 복구:** 기능 통합 과정에서 발생한 코드 유실 및 손상 문제를 `fixlog.txt`에 기반하여 최종적으로 복구함.

## 버전 11.0: 핵심 시뮬레이션 로직 개선 (상세 설정 통합)

## 버전 12.0: GUI 기반 시나리오 편집기 구현
- **요청사항:** JSON 텍스트 편집 방식 대신 GUI를 통한 시나리오 편집 기능 제공.
- **작업 내용:**
    - **변이 시나리오 편집기 (`VariantEditor`) 구현:**
        - 기존의 JSON 텍스트 편집기를 대체하는 그래픽 사용자 인터페이스 제공.
        - 변이 목록 표시, 상세 파라미터 편집, 변이 추가/삭제 기능 구현.
        - 'Day 0: Base Settings'는 메인 GUI의 '상세 설정'과 연동되며, 다른 변이들은 `variants.json` 파일에 저장/로드.
    - **개입 시나리오 편집기 (`InterventionEditor`) 구현:**
        - 기존의 JSON 텍스트 편집기를 대체하는 그래픽 사용자 인터페이스 제공.
        - 개입 목록 표시, 상세 파라미터(Day, Multiplier, Reason) 편집, 개입 추가/삭제 기능 구현.
        - 모든 개입 시나리오는 `interventions.json` 파일에 저장/로드.
